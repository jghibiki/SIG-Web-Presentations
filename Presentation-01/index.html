Git and Github

Git

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

Installing Git

Windows

Got to https://git-scm.com/ and download installer

Linux

Install via your system package manager.

Arch Linux: $sudo pacman -S git

Ubuntu: $sudo apt-get install git

Installing Git

Mac

Enable developer tools. Go to terminal and type git. This should then open a prompt asking if you wish to install dev tools.

Cloning a Git Repo

Cloning a repo pulls the remote code to your local machine

$ git clone [url to clone]

Committing Changes

  1. Clone repo
  2. Make changes to repo
  3. Add changes to staged commit
  4. Commit changes
$ git add [ file to add to commit ]
$ git commit -m "[ commit message ]"

Propogating Changes

Git, unlike SVN does not automatically push committed changes to the server. This is a much loved feature of Git.

$ git push

Github

A platform for hosting git repositories. Enables collaboration and is increadibly popular with open source projects.

Create a Github account

https://github.com/

What is a fork?

A fork is a separate copy of the original repository, that is tied to your github account.

Changes made to a fork can still be propogated into the original (called the base fork) via a Github Pull Request.

Forking a Repo

Navigate to https://github.com/jghibiki/SIG-Web-Project

Click "Fork" button

Let's clone our forks

  1. Navigate to your fork on Github.
  2. Click the clone or download button.
  3. Click the clipboard icon to copy the repository url.
  4. Go to a terminal cd to where you want to clone the repo and run $git clone [ paste repo url here ]

The DOM

HTML & DOM

HTML: HyperText Markup Language

DOM: Document Object Model

HTML

As its name states HTML is a text based markup language. It is based on XML - Extensible Markup Language

HTML is structured (like XML) via a set of tags


<html>
    <head>
         <title>App Title</title>
        <script type="text/javascript">console.log("hello")!</script>
        <link rel="stylesheet" type="text/css" href="styles.css">
    </head>
    <body>
        <h1>My Application</h1>
        <p>A paragraph!</p>
    </body>
</html>

DOM

HTML is compiled into a tree structure - a.k.a. the DOM

Rendered DOM

The DOM is then used to render the view

Manipulating the DOM

Javascript frameworks like jquery, AngularJS, and KnockoutJs. Work by allowing you to create complex applications by manipilating the DOM after it is built from the HTM "template".

This modification may involve:

  • Adding Nodes
  • Hiding Nodes
  • Removing Nodes
  • Styling Nodes

JavaScript, TypeScript and AngularJS

JavaScript

JavaScript is an implementation of the ECMAScript definition.

JavaScript 5 is the most commonly used version of JavaScript.

JavaScript Cont.

JavaScript 6 is the new version of JavaScript which implements the ECMAScript 6 definition.

Prior to ECMAScript 6, JavaScript had no truely object oriented features.

JavaScript Cont.

ECMAScript adds tons of new features:

  • Constants
  • Block Scoping for variables and functions
  • Llambda functions using double rocket notation (=>)
  • Classes
  • and more

TypeScript

AngularJS

One framework.

Mobile & desktop.

AngularJS Cont.

Encourages composable applications.

Separates logic into two realms: services and components.

AngularJS Services

  • singleton objects
  • used by the UI to get or set application state

AngularJS Components

  • Represent a moduarized portion of the UI
  • Bind to sections of the DOM and watch for changes
  • When a value on a component is changed, if that value is bound to a DOM element, the element is updated
  • The structure of a component is defined by an HTML template
